Shared-address clusters

Each group below lists two or more differently named vendors that won contracts using the same registered address. About this project

Every rupee public money spends goes through a tender. A notice is published, bids are invited, a winner is selected, a contract is signed. This process is the immune system of public finance — and it is where most large-scale procurement corruption quietly occurs. This dashboard exists to make that process auditable by anyone, not just people who can write SQL: single-bid awards, abnormally short bid windows, repeat winners, and shared-address vendor clusters are surfaced automatically, with the full source data left open for anyone who wants to dig further.

Built on publicly available government procurement data. This page reads only static JSON files generated offline — no server, no live database connection, no tracking.
